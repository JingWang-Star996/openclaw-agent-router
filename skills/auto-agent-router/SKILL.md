## 🔄 路由流程（集成实时扫描）

```
接收任务
    ↓
分析任务类型（关键词匹配 + 语义理解）
    ↓
判断：满足自动调用条件吗？
    ↓
┌───────────────────────────────────────────────────────┐
│ ✅ 满足                          │ ❌ 不满足            │
├───────────────────────────────────────────────────────┤
│ 直接调用 Agent                  │ 实时扫描 agents/    │
│                                 │       ↓             │
│                                 │ 读取 agent.md       │
│                                 │       ↓             │
│                                 │ 提取名称/职责/关键词 │
│                                 │       ↓             │
│                                 │ 智能匹配任务        │
│                                 │       ↓             │
│                                 │ 生成推荐列表        │
└───────────────────────────────────────────────────────┘
    ↓
调用 sessions_spawn 或 sessions_send / 返回推荐列表
    ↓
等待结果 → 汇总 → 返回用户
```

## 🚨 特殊情况处理

### 不确定任务类型（新增推荐机制）⭐

```
如果不满足自动调用条件，但可能有 Agent 能帮上忙：

1. 先评估：哪个/哪些 Agent 可能有帮助
2. 主动询问 + 推荐：
   "这个任务我可以自己处理，但以下 Agent 可能更专业：
    - 推荐 Agent 1：原因
    - 推荐 Agent 2：原因
    需要我调用吗？"
3. 用户确认后调用
```

**示例**：
```
用户："我想优化游戏的新手引导"

我："这个任务我可以自己处理，但以下 Agent 可能更专业：
     - AI 系统策划：专业设计新手流程
     - AI 关卡策划：设计新手关卡节奏
     - AI UX 设计师：优化新手引导体验
     需要我调用哪个？"
```

### 用户明确要求

```
用户说："你自己分析就行" → 不调用 Agent
用户说："叫 XX 来处理" → 调用指定 Agent
用户说："有没有更专业的？" → 推荐相关 Agent
```

### 紧急/简单任务

```
紧急任务（"马上修复这个 Bug"）→ 先处理，后调用
简单任务（"今天天气如何"）→ 自己处理（不推荐 Agent）
```

### 任务复杂度评估

```
简单任务（<5 分钟完成）：
- 自己处理
- 不推荐 Agent（除非用户要求）

中等任务（5-30 分钟）：
- 自己处理
- 但推荐："这个任务 XX Agent 可能更专业，需要调用吗？"

复杂任务（>30 分钟）：
- 自动调用相关 Agent
- 或推荐："这是复杂任务，建议调用以下 Agent 协作..."

## 🔧 调用方法

### 使用 sessions_spawn

```javascript
// 创建子代理会话
sessions_spawn({
  runtime: "subagent",
  agentId: "AI 数据分析师",
  task: "分析次日留存只有 30% 的原因",
  mode: "run"
})
```

### 使用 sessions_send

```javascript
// 发送到已有会话
sessions_send({
  sessionKey: "ai-data-analyst",
  message: "分析次日留存只有 30% 的原因"
})
```
